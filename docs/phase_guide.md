# 개발 단계 가이드 (Phase 1~5)

## 제품 동작 범위 (기능 정의)

사용자가 코레일 앱에서 열차 조회·필터까지 끝낸 **열차 목록 화면**을 둔 상태에서 매크로가 동작한다.

1. **새로고침**을 반복한다.
2. **보이는 행**만 대상으로 2차 필터를 적용한 뒤, `매진` → **가격(원)** 으로 바뀐 셀을 탭한다.
3. 가격 탭 후 **바텀시트(하단 시트)** 가 뜨고, 그 안의 **`예매`** 버튼을 탭한다.  
   (스크린샷 기준: 일반실 소요 시간·열차시각·운임요금·좌석선택 위에 큰 파란 **예매** 버튼.)
4. **`예매`** 탭 후 코레일이 **「승차권 정보 확인」** 화면으로 전환되면, **푸시 알림**(및 기술 스택에 정의된 진동·알람음)을 보내고 **매크로를 즉시 중단**한다. 이후 결제·입력은 사용자가 직접 한다.

**이 네 단계 + 알림이 앱의 기능 범위이다.** (목록 위 스크롤 전체 스캔 등은 별도 범위 아님.)

---

## Phase 1 — AccessibilityService 기본 구조 + UI 노드 탐색

가장 핵심이 되는 뼈대를 세우는 단계이다.

### 해야 할 작업

- `KtxAccessibilityService.kt`를 만들고 **AndroidManifest에 `<service>`로 등록**한다.  
  `BIND_ACCESSIBILITY_SERVICE`는 일반 `<uses-permission>`이 아니라, 접근성 서비스 선언·intent-filter·`android:permission` 조합으로 처리한다.
- `res/xml/accessibility_service_config.xml`에 코레일 패키지(`kr.co.korail.mobile`)와 수신 이벤트 타입(`typeWindowContentChanged`, `typeWindowStateChanged`)을 설정한다.

### 반드시 파악할 UI 트리 (코레일 실기)

**Android Studio Layout Inspector**, **App Inspection(디버깅 중 UI 계층)** 등으로 실제 기기에서 분석한다. 구형 `uiautomatorviewer`만 의존하지 않는다.

다음을 **Phase 1에서 함께** 정리해 두는 것이 좋다. (나중에 Phase 2·3에서 동일한 노드 탐색으로 이어진다.)

| 화면 | 파악할 것 |
|------|------------|
| 열차 목록 | 행이 어떤 `ViewGroup`으로 묶이는지, 열차번호·일반실/특실 가격·매진 텍스트·새로고침 버튼의 **텍스트 / contentDescription / resource-id** |
| 가격 탭 직후 | **바텀시트** 루트, **`예매`** 버튼의 텍스트·클릭 가능 노드·시트가 닫히는 조건 |
| 예매 탭 직후 | **「승차권 정보 확인」** 문자열(또는 동등한 제목) 노드 존재 여부 — 알림 트리거 판별에 사용 |

이 단계를 건너뛰면 이후 탐색 로직을 전부 다시 짤 수 있다.

### (선택) 개발 편의

Phase 4 이전에도 **최소 설정 화면**(접근성 설정 이동 링크·START/STOP·Logcat용 태그만)을 두면 Phase 2를 돌리기 쉽다. 필수는 아니다.

## Phase 2 — 새로고침 자동화 + 행 파싱 + 2차 필터 + 가격/예매 탭

앱의 실질적인 두뇌를 만드는 단계이다.

### ① 새로고침 자동 탭

`while(isActive)` 루프로 지정 주기마다 새로고침 노드를 찾아 `performAction(ACTION_CLICK)` 한다. Phase 1에서 파악한 `viewId` 또는 `contentDescription`으로 특정한다. **최소 간격은 약 2초 이상**을 권장한다(네트워크·애니메이션 여유).

### ② TrainRowParser — 행 단위 묶기

트리 전체에서 가격 노드만 찾으면 **어느 열차 행인지** 알 수 없다. 각 행(열차번호, 출발·도착, 일반실/특실 셀 등)을 하나의 그룹으로 묶는 파서가 필요하다. Phase 1의 `ViewGroup` 계층을 기준으로 행을 식별한다.

### ③ MacroFilterEngine — 2차 필터

파싱된 행에 필터(제외 열차번호, 자유석 제외 등)를 적용한다. **데이터 소스**: DataStore. Phase 4 Compose 전에는 **기본값 하드코딩** 또는 **임시 디버그 플래그**로도 개발 가능하다.

### ④ 가격 탭 → 바텀시트 내 「예매」 탭

필터를 통과한 행에서만 `매진` → 가격 패턴(예: `\d{1,3}(,\d{3})*원`)으로 바뀐 노드를 탭한다.

- 가격 탭 직후 **바텀시트가 열릴 때까지** 짧은 대기 또는 `typeWindowContentChanged`로 안정화를 기다린다.
- 시트 내부에서 **텍스트가 `예매`인 클릭 가능 노드**를 찾아 탭한다. (Phase 1에서 확인한 시트 루트 하위 탐색.)

매진→가격 감지·필터 로직은 **순수 Kotlin**으로 분리하면 단위 테스트가 쉽다.

### 테스트 전략 (이 단계)

- **TrainRowParser / MacroFilterEngine**: JVM 단위 테스트 권장.
- 코레일 실제 연동·바텀시트 타이밍: **실기 수동 검증** 위주.

## Phase 3 — 포그라운드 서비스 + 알림 + 역할 분리

매크로가 오래 켜져 있어도 끊기지 않게 하고, **승차권 정보 확인** 도달 시 사용자에게 알린다.

### 아키텍처 (권장)

- **AccessibilityService**: `rootInActiveWindow` 읽기·코레일에 대한 **클릭** 수행, `TYPE_WINDOW_STATE_CHANGED` / 콘텐츠 변경으로 **「승차권 정보 확인」** 화면 감지.
- **MacroForegroundService**(도입 시): **포그라운드 알림**(매크로 실행 중), 실행 상태 유지, **스케줄·코루틴**은 여기서 돌리되 **실제 탭은 Accessibility 쪽 API로 위임**하는 식으로 한 프로세스 안에서 역할을 나눈다. (한곳에서만 `performClick` 하도록 정리.)

### 알림

`예매` 탭 이후 **승차권 정보 확인** 전환이 감지되면 `triggerUserAlert()`로 **Push + 진동 + 알람음**(기술 스택 문서와 동일)을 보내고 **매크로 즉시 중단**한다. 알림 채널·`NotificationHelper` 소유는 구현 시 한 모듈로 모은다.

### 화면 꺼짐 방지

`FLAG_KEEP_SCREEN_ON`은 **자신의 Activity/Window**에만 적용된다. 코레일 화면을 직접 잠금 해제할 수는 없다. 필요 시 매크로 앱의 **설정/오버레이 Activity**에 적용하거나, 사용자 안내(디스플레이 타임아웃·배터리 예외)를 Phase 4·5와 함께 둔다.

## Phase 4 — Compose UI (설정·필터 화면, 로그)

- **MainScreen**: START/STOP, 새로고침 주기 슬라이더, 2차 필터(제외 열차번호, 자유석/입석 제외 등), 알림 on/off.
- **LogScreen**: 새로고침 시도, 행별 매진/가격, **바텀시트·예매 탭**, 승차권 확인 감지 등을 Flow·상태로 표시.

**선택**: `SYSTEM_ALERT_WINDOW` 플로팅 오버레이로 코레일 위에 START/STOP만 올려 UX 개선.

---

## Phase 5 — 테스트 및 안정화

### 검증 포인트

- 코레일 UI가 예상과 다를 때 **노드 미발견 시 크래시 없이** 다음 주기로 진행.
- 새로고침 주기·**바텀시트 열림 지연**에 따른 레이스(예매 버튼 미발견) — 재시도·타임아웃 정책.
- 배터리 최적화로 서비스가 죽는 경우 → **예외 설정 가이드** UI.
- 코레일 **앱 버전별** 트리 차이 — 깨지기 쉬한 순서: **행 파서 → 바텀시트·예매 → 승차권 확인 문구**.

### 테스트 범위

- 파서·필터·정규식: **자동 단위 테스트**.
- 코레일 end-to-end: **실기·수동** 또는 제한적 스모크.

## 로드맵과의 대응 (요약)

| Phase | 핵심 산출물 |
|--------|-------------|
| 1 | 서비스·XML 설정 + 목록·**바텀시트·승차권 확인** 노드 맵 |
| 2 | 새로고침 루프 + 행 파서 + 필터 + **가격 → 예매(시트 내부)** |
| 3 | FGS·알림·**승차권 확인 시 푸시·중지** |
| 4 | Compose 설정·로그·(선택) 오버레이 |
| 5 | 안정화·가이드·버전 대응 |
